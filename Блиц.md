Agents.md / системные инструкции:
— Один entry point для Codex и разработчиков (AGENTS — источник контекста).
— Документация code-first, без выдуманной архитектуры.
— Чёткое разделение current-state и target-state.
— Нереализованное нельзя описывать как существующее.
— Единая шкала статусов: implemented / partial / planned / deprecated.
— Строгий порядок чтения документации (обязателен).
— Фиксированные источники истины по коду, API, БД, интеграциям.
— Любое изменение кода → обязательное обновление соответствующих docs.
— Memory-bank как живой контекст проекта.
— PR без обновлённой документации считается неполным.
— Архитектурные границы нельзя менять молча.
— Запрет на предположения, секреты и персональные данные в docs.
— Контракты, БД и интеграции всегда синхронизированы с runtime.

UI_system.md / Дизайн инструкции
— Канонический UX/UI-стандарт для webapp, обязателен для всех PR.
— Scope строго webapp, без фреймворков, Vanilla JS + CSS.
— Telegram-native визуальный приоритет.
— Жёсткая иерархия слоёв (tokens → foundation → components → patterns → screens).
— Единый state store и единая точка HTTP для UI.
— Обязательная миграция legacy-селекторов в канонические компоненты.
— Строгие naming-конвенции (c-*, p-*, s-*, u-*, is-*).
— Только design tokens: запрет raw-цветов, размеров, отступов, радиусов, теней.
— Фиксированная spacing-scale, без произвольных значений.
— Канонический каталог UI-компонентов (MVP).
— Минимальные UX-требования (44×44, async-статусы, retry, focus, WCAG AA).
— Явные UI-контракты (JSDoc-типы).
— Жёсткий PR quality gate (syntax, скрипты, UX + A11y smoke).
— Обязательное обновление activeContext при изменении UX.
— Legacy разрешён только как compatibility, не как источник истины.
